67/*
68** Feed a token to the GLR parser.
69**
70** For each active stack head, the action table is consulted. If a
71** shift/reduce conflict exists (both actions are valid), the head is
72** forked: one copy shifts, the other reduces. After processing all
73** heads, heads that arrived at the same state are merged.
74**
75** The action table arrays are the same ones generated by Lime into
76** the parser source file.
77**
78** Parameters:
79** parser - The GLR parser context
80** token - The terminal symbol code
81** yy_action - The generated action table
82** action_count - Number of entries in yy_action
83** yy_lookahead - The generated lookahead verification table
84** lookahead_count - Number of entries in yy_lookahead
85** yy_shift_ofst - Per-state shift offset into yy_action
86** yy_reduce_ofst - Per-state reduce offset into yy_action (for goto)
87** yy_default - Per-state default action
88** nstate - Total number of parser states
89** yy_rule_lhs - LHS nonterminal for each rule (yyRuleInfoLhs)
90** yy_rule_nrhs - Negative RHS length for each rule (yyRuleInfoNRhs)
91** nrule - Number of rules
92** min_shiftreduce - YY_MIN_SHIFTREDUCE constant
93** max_shiftreduce - YY_MAX_SHIFTREDUCE constant
94** min_reduce - YY_MIN_REDUCE constant
95** max_reduce - YY_MAX_REDUCE constant
96** error_action - YY_ERROR_ACTION constant
97** accept_action - YY_ACCEPT_ACTION constant
98** no_action - YY_NO_ACTION constant
99**
100** Returns 0 on success, -1 on unresolvable ambiguity, -2 on all-heads-dead.
101*/
