1/*
2** JIT-compiled keyword tokenizer.
3**
4** Generates optimized machine code for keyword classification using LLVM
5** OrcJIT. Instead of hash table lookups, the JIT builds a trie/switch-based
6** classifier that maps input strings directly to token codes through a
7** series of character comparisons compiled to native branch sequences.
8**
9** When JIT is unavailable (LIME_NO_JIT defined), the classifier returns -1
10** for all inputs, signaling the caller to fall back to the hash-based
11** TokenTable lookup.
12*/

48**
49** Reads all keywords from the table and compiles a trie-based classifier
50** that maps (input, length) pairs to token codes. The resulting
51** JITTokenizer is independent of the original table and does not hold
52** a reference to it.
53**
54** Returns NULL if:

67/*
68** Classify a keyword using the JIT-compiled trie.
69**
70** Returns the token code if the input matches a compiled keyword,
71** or -1 if no match is found (caller should fall back to hash lookup).
72**
73** The comparison is case-insensitive for ASCII letters (matching the
74** TokenTable's behavior for SQL keywords).
75**
76** Parameters:
77** tok - JIT tokenizer (must not be NULL)
78** input - Pointer to the keyword string (not necessarily NUL-terminated)
79** len - Length of the input string in bytes
80*/
