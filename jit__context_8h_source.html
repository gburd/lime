1/*
2** LLVM OrcJIT compilation context for the extensible SQL parser.
3**
4** Provides runtime compilation of parser hot paths (action table lookups)
5** using LLVM's OrcJIT (LLJIT) infrastructure. The JIT compiles a
6** monolithic parse function that processes entire token sequences with
7** fully-inlined state dispatch, eliminating per-token function call
8** overhead.
9**
10** OrcJIT replaces the deprecated MCJIT engine and provides:
11** - Thread-safe contexts for concurrent compilation
12** - Lazy compilation support (for future tiered compilation)
13** - Better resource management and error handling via LLVMErrorRef
14**
15** When LLVM is not available at compile time (LIME_NO_JIT is defined),
16** all JIT functions degrade to no-ops and the parser falls back to the
17** standard interpreted table-driven approach.
18*/

98/*
99** Compile JIT code for a parser snapshot's action tables.
100**
101** Generates optimized machine code for find_shift_action across all
102** parser states. On success the compiled function pointers are stored
103** inside the JITContext and can be queried with jit_get_shift_action().
104**
105** The snapshot must have valid action tables (yy_action, yy_lookahead,
106** yy_shift_ofst, yy_default arrays and nstate/action_count set).
107**
108** Returns JIT_OK on success, or an error code on failure. On failure

122/*
123** Pre-warm the JIT for a set of hot parser states.
124**
125** Records which states are frequently visited so that future tiered
126** compilation can apply extra optimization to those states. Currently
127** the monolithic JIT function covers all states equally, so this is
128** a no-op beyond bookkeeping, but the API is provided for forward
129** compatibility.
130**
131** Parameters:
132** ctx - JIT context (must have compiled a snapshot)
133** hot_states - Array of state IDs to mark as hot
134** n - Number of entries in hot_states
135**
136** Returns JIT_OK on success, JIT_ERR_INVALID_ARG if ctx is NULL,
137** or JIT_ERR_NO_LLVM if compiled without LLVM.
138*/
